Technology Stack and Infrastructure

The goal of this application is the development of an MVP that helps retail investors understand the accounting entries and resource flows of basic business and investment operations. Given this focus on education and the limited resources a side project like this inherently brings with it, the technologies have been chosen based on these conditions. The following considerations were taken into account:

Security

Performance

Learning curve & development speed

Deployment

Security

The Website features both, a public area with static content like this page and a private area which can be accessed after logging in with password protected user accounts. Each user account may hold one or more roles, which determine which private pages are visible and what actions a user can take on these pages. Beyond the user name, password and the assigned roles, no further sensitive data is stored. These requirements for user authentication could be implemented via Spring Security, using JWT tokens.

Performance

The number of concurrent users logged in is not expected to surpass more than 20 at any given time, virtually removing high network traffic as a performance concern. However, each user may conduct a relatively large number of transactions which need to be appropriately tracked and executed in a timely manner. While some differences between languages do exist, most modern programming languages are able to solve the required tasks fast enough to eliminate this factor as a major decision criterion.

Learning curve & development speed

A balance needed to be found between how powerful the used technologies are and how fast they could be learned and applied to enable the development of a stable and performant with the minimum number of hours invested.

For the back-end, this implied using Spring boot which enables the creation of a production ready Java application within minutes, with minimal configuration effort. When creating a project using Spring Initializr, it is possible to choose between Maven and Gradle for build automation and dependency management which further speeds up the development process.  It also allows for the implementation of REST endpoints and DAO objects for JPA repositories, using annotations

For the front-end, the decision was somewhat more difficult given the large number of frameworks with similar specialisations. There are three names however, which take the lead  amongst established frameworks, which are Angular, Vue and React (see footnotes) [1][2][3][4].

From the perspective of requirements, any of those three frameworks would have done the job. However, Vue stands out amongst these, for being easy to learn thanks to using JavaScript rather than Typescript, having a comprehensive documentation and generally a better performance than Angular and React. Notable users of Vue.js are Alibaba, Apple and Google.                                                                                                                  

Deployment

The last few years have seen a rise of powerful and user friendly hosting solutions at competitive rates. One of these providers is Heroku by Salesforce, a "platform as a service based on a managed container system". It allows the creation of containers called "dynos", wherein each container manages the hosting of one application. Meaning that for this web application, there are two dynos: One for the Spring Boot API in the backend and one for the Vue.js application in the frontend. There is a free tier which provides all necessary functions for development and a light-weight production setup.

The deployment is quick and easy: Simply connect Your GitHub repository and ensure that any necessary environment variables are set. The initial setup for popular technologies is usually well documented thanks to the community, like here for Vue.js or directly in the Heroku Dev Center, like in this case for Spring boot. Once the GitHub account is integrated, it is possible to enable automatic deployment, allowing an update and rebuild of the application with each new GitHub commit triggered in Your IDE or using Your command line.
